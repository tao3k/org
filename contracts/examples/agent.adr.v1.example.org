* ADR: Link plans to governing specifications
:PROPERTIES:
:CONTRACT_ORG: [[../agent.adr.v1.org][agent.adr.v1]]
:ADR_ID: adr-governing-spec-links
:ADR_STATUS: accepted
:GOVERNING_REF: contracts/examples/agent.sdd.v1.example.org
:END:

** Context
Plan and ExecPlan records need to point at SDD, BDD, ADR, or PRD knowledge
without copying those documents into execution state.

** Decision
Use =GOVERNING_CONTRACT= and =GOVERNING_REF= on plan records to connect execution
state to the selected specification contract.

** Consequences
- Positive: agents can select a purpose-specific theory before planning work.
- Negative: callers must create or identify a governing spec when one is needed.
- Follow-up: new specification families should add their own contracts.

** Evidence
- The plan and ExecPlan contracts require governing specification fields.
