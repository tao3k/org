* PRD: Govern product-facing plan intent
:PROPERTIES:
:CONTRACT_ORG: agent.prd.v1
:PRD_ID: prd-govern-product-plan-intent
:PRD_STATUS: accepted
:PRODUCT_AREA: asp-org
:END:

** Problem
Agents need a product-facing requirements record when a task is about user value
or product behavior rather than architecture mechanics.

** Users
- User: agent operator reviewing why a plan exists.

** Requirements
- Requirement: product intent is recorded separately from execution steps.
- Requirement: execution plans link to this PRD when product intent governs work.

** Success Metrics
- Metric: plans can be searched by governing product requirement.

** Out of Scope
- Non-goal: this PRD does not define implementation commands or state paths.
