** What This Produces
This contract validates the shared trunk shape for ASP Org skill subtrees.
It treats Org as a stronger skill medium than Markdown: machine-readable
properties carry activation metadata, headings carry workflow phases, links
carry progressive-disclosure resources, and source blocks carry replayable
commands.

Set this contract as a package or file default with =#+PROPERTY:
CONTRACT_ORG=. Concrete skill contracts may override this trunk from a skill
subtree by setting their own =CONTRACT_ORG= property. Override contracts must
repeat any trunk requirements they still want enforced.

The shared trunk requires:
- identity metadata that can be indexed before loading the full body;
- explicit use and non-use boundaries;
- a state-oriented workflow rather than a prose-only description;
- progressive-disclosure links to resources outside the skill body;
- validation or pressure-test guidance;
- adversarial review criteria for content quality that cannot be fully checked
  by structural assertions;
- provider-owned commands in shell source blocks;
- package-relative links, never install-root placeholders or repo-root paths.
