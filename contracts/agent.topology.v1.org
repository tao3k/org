* agent-topology-v1

:CONTRACT_ID: agent.topology.v1
:CONTRACT_SCOPE: subtree
:CONTRACT_KIND: org-elements
:TITLE: Agent Project Topology v1
:END:

** What This Produces
This contract validates the coarse navigation layer for an imported project, not
the detailed Org memory/session ledger.

The intended output is a project topology root that records:
- the imported project identity;
- the topology scope and durable topology id;
- a coarse node kind such as project, root session, child session, content
  anchor, context pack, or sub-agent route;
- a coarse edge kind such as imports-project, opens-session,
  branches-from-session, forks-from-content, compresses-to-context-pack, or
  routes-to-subagent;
- a source anchor back to the detailed Org fact layer;
- Overview, Navigation, and Visibility Boundaries sections.

Topology is the route map. Detailed content, claims, source spans, and session
records remain in the Org memory/session contracts and are reached through the
source anchor.

See =org/contracts/examples/agent.topology.v1.example.org= for the concrete
topology shape this contract accepts.

** topology-has-id-property
:PROPERTIES:
:ASSERT_ID: topology.has-id-property
:SEVERITY: error
:END:

#+BEGIN_SRC org-contract
let topology = headline where at($scope)
assert count node-property where descendant_of(topology) and summary(key) = "TOPOLOGY_ID"
#+END_SRC

#+BEGIN_SRC jinja2 :name message
Topology `{{ scope.title }}` must carry a stable TOPOLOGY_ID property.
#+END_SRC
