* agent-plan-v1
:PROPERTIES:
:CONTRACT_ID: agent.plan.v1
:TITLE: Agent Plan Ledger v1
:END:

** What This Produces

This contract validates a recoverable plan ledger. It defines the theory of a
plan record: a bounded objective, a scope, a governing specification reference,
recoverable state, evidence, and a next action. Runtime commands, storage paths,
and client-specific workflows are out of scope for this contract.

The intended output is a single recoverable task heading with:

- a native Org workflow state and progress cookies;
- identity, objective, scope, governing specification, status, and recovery properties for later agents;
- a lifecycle checklist that shows where the plan stands;
- Context, Checkpoints, Validation, Evidence, and Recovery sections;
- concrete evidence and recovery anchors that can be inspected without a live LLM.

** plan-has-workflow-state
:PROPERTIES:
:ASSERT_ID: plan.has-workflow-state
:SEVERITY: error
:END:

#+BEGIN_SRC org-contract
(assert count >= 1
  (and
    (headline :at $scope)
    (or
      (= (property :todo-type) "todo")
      (= (property :todo-type) "done"))))
#+END_SRC

#+BEGIN_SRC jinja2 :name message
Plan `{{ scope.title }}` must be an Org task heading with a TODO/DONE workflow state.
#+END_SRC
