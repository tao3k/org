:CONTRACT_ALIAS: ../contracts/skill.v1.org#skill.v1 ../contracts/skill.v1.org
:CONTRACT_KIND: org-elements
:CONTRACT_SCOPE: subtree
:END:

** What This Produces
This contract validates the shared shape for Org-backed skill subtrees.
It treats Org as a stronger skill medium than Markdown: machine-readable
properties carry activation metadata, headings carry workflow phases, and source
blocks carry replayable commands.

Apply this contract directly to any Org skill subtree with a =CONTRACT_ORG=
property. Product- or provider-specific skill contracts should be listed as
additional =CONTRACT_ORG= properties on the same subtree instead of pretending
that contract inheritance is implicit.

The shared skill contract requires:
- identity metadata that can be indexed before loading the full body;
- explicit use and non-use boundaries;
- a state-oriented workflow rather than a prose-only description;
- validation or pressure-test guidance;
- adversarial review criteria for content quality that cannot be fully checked
  by structural assertions;
- provider-owned commands in shell source blocks.
