** What This Produces
This contract validates a recoverable execution plan for large, risky, or
cross-cutting agent work. It is intentionally separate from =agent.plan.v1=:
=agent.execplan.v1= governs the plan before and during execution, while
=agent.plan.v1= governs the task ledger an agent leaves after work is done.

The intended output is an Org document with:
- an ExecPlan root heading;
- explicit scope and boundaries;
- a self-check before widening work;
- context, work plan, concrete steps, validation, audit, and final gate sections;
- replayable evidence expectations without embedding client-specific behavior.
