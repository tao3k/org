* ASP TaskFrame Archive Scenario

** What This Reproduces

This scenario defines how a completed, suspended, superseded, or handed-off ASP
task keeps its useful evidence without preserving noisy transient context. It
connects Org-readable archives with JSON packets so humans, agents, Python graph
analysis, and Julia research notebooks can inspect the same task history.

The archive is not a memory dump. It is a compact task-local evidence boundary:

- stable facts can be promoted into project evidence
- volatile dirty-overlay evidence expires with the task frame
- observations preserve why the model changed course
- promoted facts feed future search ranking, topology jumps, and policy
  selection

** Promotion Rules

Promoted facts must be:

- stable across task frames
- tied to selectors or artifact ids, not line numbers
- backed by observations, snapshots, benchmark receipts, scenario receipts, or
  prior failure evidence
- assigned an owner and expiration policy

Facts that only describe a temporary dirty diff, a one-off model hypothesis, or
an unverified local guess stay inside =ephemeralEvidence=.

** Graph And Research Use

Python can consume the JSON archive as a graph input:

- task frame nodes connect intent, feature, selectors, policy facts, scenarios,
  observations, and promoted facts
- topology routing can rank future jumps by successful prior paths
- failure nodes can be used as negative examples for ranking and policy
  selection

Julia can consume archive snapshots offline:

- replay many sessions
- evaluate which topology jumps reduced turns
- find ranking failures and feed corrected graph features back to Python
- compare scenario-backed policy selection against generic policy selection

Julia does not own runtime search. It owns algorithm research and evidence
analysis.

** Prompt Budget Rule

An archive render should give the agent a compact resume surface:

- current status
- feature topology id
- selector list
- policy/scenario facts actually used
- observations that changed the plan
- next command or handoff note

Full packet lists and source excerpts stay behind ids.

** Failure Modes

- If archive stores every transient packet without promotion rules, the scenario
  fails.
- If stable facts cannot be linked back to source artifacts, the scenario fails.
- If Org and JSON disagree about task status, the scenario fails.
- If future ranking cannot tell promoted facts from expired dirty overlay facts,
  the scenario fails.
- If the archive is only useful to humans and not to Python/Julia analysis, the
  scenario fails.
