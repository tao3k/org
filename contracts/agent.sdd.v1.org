#+TITLE: Agent SDD Contract v1

* agent-sdd-v1
:PROPERTIES:
:CONTRACT_ID: agent.sdd.v1
:CONTRACT_KIND: org-elements
:CONTRACT_SCOPE: document
:END:

** What This Produces
This contract validates a software design description. An SDD records durable
architecture knowledge: system boundaries, quality attributes, design concerns,
and links that execution plans can use as governing context. It does not track
implementation progress, commands, or agent state.

Use =agent.sdd.v1= as the governing contract when a plan or ExecPlan is driven
by architecture, migration, integration, runtime, data, deployment, or design
drift concerns.
