** What This Produces
This contract validates a behavior specification. A BDD record captures a
feature, actors, scenarios, and acceptance evidence. Use it as a governing
contract when a plan is mainly about user-visible behavior or acceptance tests.

** bdd-has-feature-root
:PROPERTIES:
:ASSERT_ID: bdd-has-feature-root
:SEVERITY: error
:END:

#+BEGIN_SRC org-contract
(assert exists
  (headline :summary-contains (title "Feature")))
#+END_SRC

#+BEGIN_SRC jinja2 :name message
BDD files must have a Feature root heading.
#+END_SRC
