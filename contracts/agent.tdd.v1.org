:CONTRACT_KIND: org-elements
:CONTRACT_SCOPE: document
:END:

** What This Produces
This contract validates a test-driven development specification. A TDD record
captures the target under test, the expected failure-first test, the
red-green-refactor loop, and regression evidence. Use it as a governing contract
when a plan is mainly about test-first implementation, regression protection, or
executable test contracts.

** tdd-has-root
:PROPERTIES:
:ASSERT_ID: tdd-has-root
:SEVERITY: error
:END:

#+BEGIN_SRC org-contract
(assert exists
  (headline :summary-contains (title "TDD")))
#+END_SRC

#+BEGIN_SRC jinja2 :name message
TDD files must have a TDD root heading.
#+END_SRC
