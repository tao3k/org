* agent-evidence-loop-v1
:PROPERTIES:
:CONTRACT_ID: agent.evidence-loop.v1
:CONTRACT_SCOPE: subtree
:CONTRACT_KIND: org-elements
:END:

** What This Produces
This contract validates an evidence loop receipt. A receipt ties an agent claim or reflection answer to parser-visible evidence, a replayable command, and an evaluated result marker. The contract does not execute commands; it verifies that the Org record contains the evidence graph anchors needed for a later runner or reviewer to replay the command.

The expected shape is an =Evidence Loop= subtree with:
- an =EVIDENCE_LOOP_ID= property;
- an =EVAL_STATUS= property set to =passed=;
- an Org table with =Claim=, =Evidence=, =Command=, and =Result= columns;
- at least one non-header =Evidence= cell containing a real Org link;
- at least one non-header =Command= cell and one shell source block;
- at least one non-header =Result= cell.

** evidence-loop-has-id
:PROPERTIES:
:ASSERT_ID: evidence-loop-has-id
:SEVERITY: error
:END:

#+BEGIN_SRC org-contract
(assert exists
  (node-property :descendant-of $scope :summary (key "EVIDENCE_LOOP_ID")))
#+END_SRC

#+BEGIN_SRC jinja2 :name message
Evidence loop receipts must declare EVIDENCE_LOOP_ID.
#+END_SRC
