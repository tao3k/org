** System Boundary
Describe the system, the boundary it owns, and the external systems or
providers it relies on.

** Quality Attributes
- Quality: correctness
- Quality: maintainability
- Quality: performance

** Design Concerns
- Concern: describe a durable architecture concern.
- Constraint: describe a boundary that implementation plans must preserve.

** Architecture Views
*** Runtime View
Describe runtime components and interaction boundaries.

*** Data View
Describe stored facts, identifiers, and ownership boundaries.

** Decisions
*** Decision: title
State the selected design direction and rejected alternatives.

** Fitness Criteria
- Gate: record the measurable condition for accepting implementation work.
- Drift: record what evidence should trigger redesign.

** Linked Implementation
Implementation work belongs in an Org task or ExecPlan that uses
