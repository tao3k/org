** Test Intent
Describe the behavior, module, or regression surface that should be proven by tests before implementation is accepted.
** Red Green Refactor Loop
- Red :: name the failing test or missing assertion expected first
- Green :: name the minimal implementation path that should make the test pass
- Refactor :: name the cleanup boundary allowed after green
** Test Cases
| Case | Expected Failure | Passing Evidence |
| primary behavior | pending failing test | pending |
** Regression Evidence
- Record the command, status, and short result proving the test contract.
