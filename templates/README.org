* Templates
This directory contains language-facing Org authoring templates. Provider
crates such as =languages/orgize= implement parsing and evaluation; reusable
agent templates and their public contracts live here.

** Contract And Template Change Workflow

Contracts and templates are a paired surface. When a file under
=languages/org/contracts/= changes required properties, headings, file naming,
storage placement, validation sections, or recovery semantics, update the
corresponding file under =languages/org/templates/= in the same change.

Use this workflow:

- Identify the contract id and paired template from this README before editing.
- Update the contract first when the contract semantics are changing.
- Update the paired template so a freshly rendered entry satisfies the new
  contract without manual repair.
- Update this README if the pair, filename convention, storage convention, or
  authoring workflow changes.
- Run =asp org contract trace --org-contract-registry ../contracts/<id>.org <template-file>=
  from =languages/org/templates=, or the equivalent workspace-relative command,
  and record the result with the change.

Do not leave a contract-only change that makes the paired template stale. If a
contract change intentionally has no template effect, state that explicitly in
the change summary.

** Basic Template Metadata
:PROPERTIES:
:TEMPLATE_ID: basic.template.v1
:TEMPLATE_KIND: base
:TEMPLATE_FILE: basic.template.v1.org
:TEMPLATE_CONTRACT: basic.template.v1
:END:

- Template: [[basic.template.v1.org][basic.template.v1.org]]
- Contract: [[../contracts/basic.template.v1.org][basic.template.v1]]
- Example: [[../contracts/examples/basic.template.v1.example.org][basic.template.v1.example.org]]
- Multi-contract Example: [[../contracts/examples/basic.template.v1.with-task.example.org][basic.template.v1.with-task.example.org]]

Use this template when an Org document needs reusable document metadata before
it is combined with a purpose-specific contract. The paired contract verifies
=#+TITLE=, =#+DATE=, =#+AUTHOR=, =#+FILETAGS=, document-level =#+PROPERTY=
metadata, and =#+CONTRACT_ORG=. The template date value uses a
seconds-precision timestamp shape such as =2026-06-24T12:34:56Z=. Orgize can
evaluate multiple =#+CONTRACT_ORG= declarations, so another contract can be
added only when the document actually needs that contract's semantics.

** Plan

:TEMPLATE_KIND: plan
:TEMPLATE_FILE: agent.plan.v1.org

:END:

- Template: [[agent.plan.v1.org][agent.plan.v1.org]]
- Contract: [[../contracts/agent.plan.v1.org][agent.plan.v1]]

Use this template when an agent needs to leave a recoverable plan ledger after
work is done. The paired contract verifies identity, objective, scope, status,
recovery, lifecycle, validation, and evidence sections. Specification references
are added only when the task actually needs them.

Use these templates when work needs a governing specification before or
alongside planning. SpecFlow covers the process of adding a new specification
family. SDD covers software architecture and design drift, ADR covers a specific
decision, BDD covers behavior and acceptance scenarios, TDD covers test-first or
regression contracts, BDR covers Business Development Representative workflows,
and PRD covers product requirements.
