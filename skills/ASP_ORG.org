** Use Boundary
Use this skill when the task depends on durable Org planning state, Org
contracts, Org templates, TODO headlines, checklist items, property drawers,
planning timestamps, or source blocks that should be queried through =asp org=.

Do not use this skill when the task is generic Markdown authoring, free-form
chat planning, or client-local todo management that does not need Org AST facts
or Org contract validation.

** State Workflow
Start by asking the provider what Org facts exist, then let the result shape the
next action. The provider exposes asp org query for semantic AST facts. The provider exposes asp org contract trace for contract validation.

#+begin_src shell
asp org guide
asp org query --kind task --field todo=TODO --workspace . --view metadata
asp org query --kind checklistItem --field checked=false --workspace . --view metadata
asp org query --kind property --field key=<KEY> --workspace . --view metadata
#+end_src

- If the query returns TODO headlines, treat them as =kind=task= and inspect
  the relevant selector before editing.
- If the query returns checkbox list items, treat them as =kind=checklistItem=;
  do not collapse them into task headlines.
- If the query returns a property drawer match, use the property key/value as
  the semantic anchor; do not invent SDD, Wendao, or agent-plan search modes.
- If no query result matches, narrow the AST query or inspect the linked
  contract/template before creating new structure.

Validate contract or template state through the contract provider, not by
reading prose alone:

#+begin_src shell
asp org contract trace --org-contract-registry ../contracts/agent.execplan.v1.org ../templates/agent.execplan.v1.org
asp org contract trace --org-contract-registry ../contracts/asp.skill.trunk.v1.org --org-contract-registry ../contracts/asp.skill.v1.org ../skills/ASP_ORG.org
#+end_src

** Plan Capture
Use =capture-plan= only for a non-mutating, reviewable Org entry plan. Capture
kinds stay generic; plan contracts are caller-supplied Org properties and body
content.

#+begin_src shell
asp org capture-plan \
  --kind task \
  --title <TITLE> \
  --target-file PLANS.org \
  --outline Plans/Active \
  --tag plan \
  --property PLAN_ID=<PLAN_ID> \
  --body <OBJECTIVE>
#+end_src

The command renders an Org entry and application preconditions. It does not
write the target file.

** Rules
- Preserve the semantic AST provider boundary: query Org tasks, checklist
  items, properties, planning timestamps, links, and source blocks as Org facts.
- Do not use =--kind agent-plan=; =agent-plan= is not a provider-owned capture
  kind.
- Do not reintroduce legacy guide labels such as =agent-plan-*=,
  =wendao-task-*=, =wendao-orgid-*=, =sdd-property=, or =plan-record=.
- Do not encode SDD, Wendao, or task archive behavior as hidden search/query
  side effects. Express them as AST queries first, then route mutations through
  an edit-plan or AST-patch contract.
- Do not rewrite package-relative links as installation-root placeholders or
  repository-root paths.

asp org query --kind property --field key=RECEIPT_STATUS --field value=pending --workspace . --view metadata
